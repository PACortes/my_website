1. Definición

Consola

Es el corazón de Ry RStudio. Aquí escribiremos las lineas de comando para “pedir” cosas a RStudio. En la consola también visualzaremos los resultados de las petición realizada (excepto gráficos). Es importante tener en cuenta que:

  • Cuando la consola tiene el cursor > significa que le podemos dar comandos para ejecutar.
  • Si es que tiene el símbolo + quiere decir que nos falta completar el comando anterior.
  • El texto en rojo nos avisa y explica, errores en la solicitud realizada.

Script

Es un documento de texto con la extensión de archivo .R que puede ser leido por R.

En el script escribiremos y ejecutaremos nuestras líneas de código. Además, en el script podemos escribir notas personales. Es importante tener en cuenta que:

  • Para ejecutar una(s) línea(s) de comandos, debemos presionar <Ctrl + R> (Windows) o <Ctrl +Enter> (Mac). Los resultados se visualizaran en la consola.

  • Para geneerar notas personales, las(s) línea(s) de texto y/o códigolaLas deben estar precedidas por el el símbolo #.

2. Tipos de datos en R

Los tipos de datos de uso más común en R son:

Numérico

Representan números que tienen una parte decimal o fraccionaria.

5
## [1] 5

25.3
## [1] 25.3

123890.85544
## [1] 123890.9

Cadena de texto

Secuencias de caracteres. Puede contener letras, números, espacios, signos de puntuación y símbolos especiales.

"a"
## [1] "a"

"a"
## [1] "a"

"Contaminacion"
## [1] "Contaminacion"

Factor

Dato numérico representado por una etiqueta.

Lógico

Sólo tienen dos valores posibles: verdadero (TRUE) y falso (FALSE). Representan si una condición o estado se cumple, es verdadero, o no, es falso.

Perdido (NA) y Vacío (NULL)

En R, usamos NA para representar datos perdidos, mientras que NULL representa la ausencia de datos. La diferencia entre las dos es que un dato NULL aparece sólo cuando R intenta recuperar un dato y no encuentra nada, mientras que NA es usado para representar explícitamente datos perdidos, omitidos o que por alguna razón son faltantes.

Verificando el tipo de dato

Podemos verificar si un dato es de un tipo específico con la familia de funciones is(). Estas funciones toman como argumento un dato, si este es del tipo que estamos verificando, nos devolverán TRUE y en caso contrario devolverán FALSE

#library(tableHTML)
library(gt)
tabla <- data.frame(
  FU=c("is.integer()","is.numeric","is.character()","is.factor()","is.logical()","is.na()","is.null()" ),
                 TD=c("Entero","Numérico","Cadena de texto","Factor","Lógico","NA","NULL"))

tabla %>%
  gt() %>% 
  cols_label(FU="Función",
             TD= "Tipo de dato") %>%
  tab_style(locations=cells_column_labels(columns=everything()),
            style = list(
              cell_borders(sides="bottom",weight=px(3)),
              cell_text(weight = "bold"))) %>%
  tab_options(table.font.size = px(20L),
              column_labels.font.size = px(25L))
Función Tipo de dato
is.integer() Entero
is.numeric Numérico
is.character() Cadena de texto
is.factor() Factor
is.logical() Lógico
is.na() NA
is.null() NULL
