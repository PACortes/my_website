R Markdown

Rmarkdown nos permite generar informes a partir de R. Utiliza un lenguaje llamado Markdown para enriquecer el informe final. Este permite incluir texto, comandos de R, imágenes y gráficos a un documento. Pero lo más importante es que permite que se reproduzca el análisis realizado y si se incorporan nuevos datos, los resultados se actualizarán. Para mas detalles visitar http://rmarkdown.rstudio.com.

1. ¿Como funciona?

Debemos comenzar abriendo un archivo R Markdown, un archivo de texto sin formato, que tiene la extensión .Rmd.

En el menú debemos hacer click en: File –> New File –> R Markdown…

Esto abrirá un fichero que nos permitirá elegir el formato de salida de nuestro informe:

  • página web que se podrá ver con un navegador,
  • documento PDF o
  • documento Word.

Importante: Si deseamos generar una documento PDF, necesitaremos instalar LaTeX. Para los usuarios de Markdown R que no hayan instalado LaTeX anteriormente, se recomienda instalarTinyTeX (https://yihui.name/tinytex/):

Encabezado YAML

El encabezado YAML (en inglés yet another markup language) determina los parámetros generales de formato para el documento a generar. Es importante tener presente que este encabezado define el formato del archivo resultante, más no su contenido.

El encabezado comienza y termina con tres rayas (- - -). Veamos el siguiente ejemplo que incluye algunos parámetros (existen muchos más) que definirán el formato del documento:

---
title: "Título del documento"
author: "Quien desarrolla el documento"
date: "Fecha en la cual desarrollamos el documento"
output:
  html_document: default
---

En este ejemplo hemos especificado que el documento se generará en formato HTML. Otras opciones de output son:

  • output: html_document… archivo html (página web)
  • output: pdf_document… archivo pdf
  • output: word_document… Microsoft Word .docx
  • output: beamer_presentation… presentación beamer (pdf)
  • output: ioslides_presentation… presentación ioslides (html)

Texto

El documento lo podemos escribir en forma de texto plano (el contenido que deseas escribir). Para ello podemos usar la sintaxis de markdown para describir el formato del texto en el documento final. Dedicaremos una sección independiente para revisar la sintaxis de RMarkdown.

Podemos agregar imágenes a un informe de R Markdown a partir de imágenes disponibles en nuestro directorio de trabajo. Para ello utilziaremos la siguiente sintaxis:

Trozos (chunk) de código

Los trozos (chunk) de código en un documento de R Markdown contendrán tu código R. Los trozos de código está delimitado por tres apóstrofes seguidos por un r entre corchetes curvos {r}, y su cierre por otros tres apóstrofes (```). Eso delimita lo que se ejecutará como código.

Un chunk básico se ve de la siguiente forma:


# ``` {r } 

#Aquí va el código

# ```

Veamos un ejemplo:

# sumemos
x <- 1+12

y <- 90

# generemos un objeto que contenga la resta de "x" con "y" 
respuesta <- x - y

# revisemos el resultado de nuestro objeto llamado respuesta
respuesta
## [1] -77

Ahora, insertemos un nuevo chunk. Para ello podemos escribirlo o podemos utilizar una combinación desde el teclado para generar un chunk por defecto:

  • Windows: Ctrl + Alt + I
  • OS X: Cmd + Option + I
#```{r }

#```

RMarkdown también nos permite configurar qué es lo que queremos mostrar en nuestro documento final. Por defecto, mostramos el código y resultado. Por ejemplo, al abrir el siguiente código de nuestra sintaxis de Markdown con las siguintes opciones {r, eval = FALSE, echo = FALSE, message = FALSE, warning= FALSE} les estamos indicando lo siguiente:

  • eval = FALSE: queremos mostrar el código, pero que no se ejecute (sólo queremos enseñar cómo se hace algo).
  • echo = FALSE: queremos mostrar el resultado pero no el código (no queremos enseñar cómo se hace algo).
  • message = FALSE: No muestra los mensajes de información que se generan al correr el código.
  • warning = FALSE: No muestra los mensajes de advertencia que se generan al correr el código.

4. Syntaxis

La syntaxis de RMarkdown es muy secilla y nos permite añadir estilo, imágenes, enlaces y expresiones mátematicas a nuestro documento de texto. Las palabras o frases deben comenzar y terminar con los siguientes simbolos:

Syntaxis Resultado
* * cursiva
_ _ cursiva
** ** negrita
__ __ _negrita
superindice ^ ^ superindice 2
tachado ~~ ~~ tachado
enlace [ ] [https://pacortes.github.io/]

También podemos generar listas no numeradas:

  • Item 1
  • Item 2
    • Item 2a
    • Item 2b

y listas numeradas:

  1. Item 1
  2. Item 2
  3. Item 2
    • Item 3a
    • Item 3b

En algunas ocasiones, estaremos interesados en establecer secciones dentro de nuestro documento. Para ello podemos diferenciar mmúltiples encabezados, según el tamaño del texto.

Nivel Sintaxis
Encabezado 1 # texto

Encabezado 2 |## texto

Encabezado 3 |### texto

Encabezado 4 |#### texto

Encabezado 5 |##### texto

Encabezado 6 |###### texto
